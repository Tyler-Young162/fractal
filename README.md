# 分形创世实验舱 — 科技馆互动展品

## 项目概述

这是一个部署在**科技馆触控大屏**上的互动展品：观众通过选择基础几何图形、调节分形参数，实时生成绚丽的分形艺术图案。

**核心教育目标**：让观众直观理解"分形"——简单的规则经过递归/迭代可以产生无穷复杂的美。

---

## 当前技术栈

| 层面 | 选型 | 说明 |
|------|------|------|
| 渲染引擎 | **p5.js 1.9** (Canvas 2D) | 轻量、零依赖，适合科技馆长期无人值守运行 |
| UI | 原生 HTML/CSS | 左侧全高毛玻璃面板，参数即时生效 |
| 部署方式 | 单个 `fractal_demo.html` | 浏览器直接打开即可运行，无构建步骤 |

> **如果后续效果复杂度提升**，可考虑迁移渲染层到 **WebGL (Three.js / pixi.js / raw shader)**，以获得 GPU 加速、后处理特效（Bloom/Glow）和更高帧率。

## 当前已实现的功能

### 1. 基础图形库（8 种）

| ID | 名称 | 绘制方式 |
|----|------|---------|
| `triangle` | 三角形 | 正三角 polygon(3) |
| `square` | 正方形 | rectMode(CENTER) |
| `circle` | 圆形 | circle() |
| `pentagon` | 五边形 | polygon(5) |
| `hexagon` | 六边形 | polygon(6) |
| `star5` | 五角星 | 内外径交替顶点 |
| `cross4` | 四芒星 | 极窄内径的 4 尖星 |
| `flower` | 樱花瓣 | `r = base + sin(5θ) * amp` |

### 2. 图形组合机制

- 支持**多选**图形（至少保留 1 个）
- 选中的图形按顺序排入一个**可拖拽排序的队列**
- 分形递归时，按队列顺序在**每一层交替使用不同图形**
  - 例如队列 `[三角形, 圆形, 五角星]`：第 0 层三角 → 第 1 层圆 → 第 2 层星 → 第 3 层又回到三角...

### 3. 分形算法参数

| 参数 | 变量名 | 范围 | 说明 |
|------|--------|------|------|
| 繁衍深度 | `params.depth` | 0–7 | 递归层数，指数级影响性能 |
| 基因收缩率 | `params.scale` | 0.20–0.75 | 子代与母体的尺寸比 |
| 分裂数覆写 | `params.branches` | 0–12 (0=自动) | 覆写图形自然顶点数，产生异形分裂 |
| 层间旋转角 | `params.layerRot` | 0–180° | 每层额外旋转，产生螺旋/曼荼罗 |
| 螺旋因子 | `params.spiral` | -1.0–1.0 | 子代位移半径随层级递进偏移 |
| 扩散半径比 | `params.spread` | 0.3–2.0 | 子代离母体中心的远近倍率 |
| 中心子代 | `params.center` | bool | 母体中心额外生成缩小副本 |
| 交替镜像 | `params.mirror` | bool | 奇偶层水平翻转 |
| 自转速度 | `params.spin` | 0–2.0 | 整体画面旋转速度 |
| 呼吸脉动 | `params.pulse` | 0–0.3 | 各层尺寸随时间正弦跳动 |
| 色相偏移 | `params.hueShift` | 0–60 | 每层色相偏转角，控制彩虹程度 |

### 4. 色彩系统

- 使用 **HSB 色彩空间**
- 观众选一个主色相，系统自动按 `hueShift` 每层偏转
- 亮度随层级递增（越深越亮 → "自发光"视错觉）
- 提供 10 色快捷调色盘

## 后续深化方向（建议）

以下是可以继续优化的方向，**算法层面不需要严格遵守当前的递归 IFS 框架**，可以大胆尝试不同的分形生成方法。最终目标只有一个：**视觉效果要震撼、要美**。

### 🎨 视觉效果增强

1. **发光与辉光 (Bloom/Glow)**
   - 当前 Canvas 2D 没有后处理管线。最理想的方案是迁移到 WebGL，利用 off-screen framebuffer 做高斯模糊叠加实现真正的 bloom
   - 轻量替代方案：用 CSS `filter: blur()` + 叠加层模拟外发光

2. **渐变填充与纹理**
   - 当前图形是纯色块。可以给每个图形改用径向渐变 (`createRadialGradient`)，或者用 `blendMode(ADD)` 做加法混合让重叠区域更亮
   - 考虑给深层图形加上描边 (`stroke`) 而非纯填充，会产生线稿风分形

3. **粒子点缀**
   - 在分形的最深层（叶节点）释放微小粒子，模拟"能量溢出"，增加画面的生命感
   - 粒子可以沿着分形的枝干路径运动

4. **背景氛围**
   - 当前背景是静态深色。可以加入缓慢运动的星云/粒子场作为背景层
   - 或者让背景色随主色相联动，形成统一的色调氛围

### 🧬 算法多样性

当前只实现了一种算法（顶点位置递归放置缩小副本 = 经典 IFS）。可以大幅扩展：

1. **L-System (林德迈耶系统)**
   - 基于字符串重写规则的分形，特别适合生成植物、树木、科赫雪花
   - 例如科赫雪花规则：`F → F+F--F+F`，每次将直线段替换成折线
   - 可以让用户自定义重写规则（高级模式）

2. **逃逸时间分形 (Mandelbrot / Julia 集)**
   - 在复数平面上迭代 `z = z² + c`，根据逃逸速度着色
   - 视觉冲击力极强，可以做无限缩放动画
   - 这类分形不使用"基础图形"，但可以作为一个独立的分形算法模式

3. **混沌游戏 (Chaos Game)**
   - 在多边形顶点之间随机跳跃取中点，积累大量点形成分形
   - 非常适合做"逐点生长"的动画过程，教育意义极强
   - 三角形 → 谢尔宾斯基三角，正方形 → 谢尔宾斯基地毯

4. **分形树 (Fractal Tree)**
   - 每个枝干分裂成 N 个子枝干，角度和长度可调
   - 可以和风力扰动结合，做出随风摇摆的分形森林

5. **阿波罗尼奥斯垫片 (Apollonian Gasket)**
   - 圆的递归相切填充，纯圆形的分形，数学上非常优美

### 🖥️ 交互体验升级

1. **预设方案快捷切换**
   - 提供 5–10 个精心调好的预设（如"宇宙晶花"、"深海珊瑚"、"曼荼罗法阵"），一键切换所有参数
   - 观众探索完预设后再自己微调，降低上手门槛

2. **无人值守吸引模式 (Attract Mode / Screensaver)**
   - 30 秒无操作后自动进入：系统随机切换预设、缓慢变化参数
   - 新观众触摸屏幕时自动退出，恢复交互模式

3. **截图保存 / 二维码带走**
   - 观众满意当前图案后，点击"保存"生成一张高清截图
   - 屏幕上弹出二维码，观众扫码即可把自己创作的分形带走

4. **触屏手势优化**
   - 双指缩放/旋转画面（科技馆为触控屏）
   - 单指拖动平移画面
   - 所有滑块增大可触区域（适配粗手指操作）

### ⚡ 性能与稳定性

- **深度 ≥ 6 时的性能警告**：当前递归 n=7 在某些图形组合下（如六边形 × 高分裂数）可能产生百万级绘制调用。如果要开放更高深度，必须引入：
  - 节点数预估计算 + 动态深度熔断
  - 或改用 WebGL instanced rendering
- **内存泄漏防护**：p5.js 长时间运行需注意 canvas context 状态栈的 push/pop 平衡
- **崩溃自恢复**：科技馆展品建议外层套一个进程守护（如 `pm2` 或系统级 watchdog），浏览器崩溃后自动重启

## 关键代码入口

打开 `fractal_demo.html`，核心逻辑集中在 `<script>` 标签内：

- **`setup()`**：初始化画布、绑定 UI 事件
- **`draw()`**：每帧调用，清除背景 → 缓动插值 → 调用 `drawFractal()`
- **`drawFractal(x, y, radius, depthRemaining, scale, baseHue, layer)`**：**递归核心**，所有分形算法参数在此生效
- **`renderShape(shape, radius)`**：根据 shape ID 分发到具体的绘制函数
- **`renderQueue()` / 拖拽事件系统**：管理图形的选择与排序队列
